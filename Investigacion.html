JavaScript


¿Qué es JavaScript?

JavaScript es un lenguaje de programación o de secuencias de comandos que te permite implementar funciones complejas en páginas web, cada vez que en una página web hace algo más que sentarse allí y mostrar información estática para que nosotros podamos visualizarla, sus oportunas actualizaciones de contenido, mapas interactivos, animación de gráficos 2D/3D, desplazamiento de máquinas reproductoras de video, etc.

Además JavaScript esta involucrado en lo que seria HTML y CSS que van de la mano como parte de las tecnologías web estándar; y es muy sorprendente como se superponen demasiado bien, Por lo que llegados a esto hay que preguntarnos algo muy importante.


¿Para qué sirve JavaScript?


En la actualidad JavaScript se utiliza para todo, gracias a la itroducción de Node.js. Esta tecnología crea software robusto para empresas de todo el mundo. Por si fuera poco, organizaciones como LinKedIn y Medium lo implementan al construir plataformas para que los usuarios tengan acceso a sus servicios.

Lo que se puede hacer con JavaScript abarca diferentes tipos de software, como juegos, programas de computadora, aplicaciones web y hasta tecnologías de blockchain. JavaScript es posiblemente el lenguaje de programación mas popular de la web.


¿Cómo utilizar JavaScript en una página web?


El uso más popular para JavaScript seria para el desarrollo web, y es una de las herramientas más poderosas que un desarrollador puede tener en sus manos. Los desarrolladores usan JavaScript en esta área para añadir interactividad y funciones que mejoren las experiencias que recibe el usuario y que internet disfrute más.

Además JavaScript se ha expandido más allá del desarrollo de interfaz, que es donde comenzó; JavaScript ha logrado llegar al back-end, o dorsal de desarrollo web. Esto quiere decir que los desarrolladores tienen acceso de interfaz a métodos CRUD(Create, Read, Update, Destroy; en español: Crear, Leer, Actualizar, Destruir) y hasta puede utilizarse en el motor de un sitio web. Convirtiendolo en algo primordial en la tecnología de desarrollo web.


Algunos usos específicos de JavaScript en desarrollo web:


•Interactividad de interfaz o frontend: mejora lo que vemos los usuarios a través de la pantalla siendo los elementos visuales como botones o gráficos, permitiendo a los usuarios interactuar con la aplicación o sitio.

•Aplicaciones web: las aplicaciones web son similares a los sitios, pero en su lugar pueden empacarse en una caja mas compacta, que mejora el control de la seguridad y otros aspectos.

•Juegos de navegador: los navegadores web actuales han cambiado mucho; los desarrolladores pueden crear juegos robustos que funcionan en ellos.

•Desarrollo web dorsal o back-end: el desarrollo web se ha transformado tanto, que JavaScript puede utilizarse para gestionar los datos e infraestructuras que permiten que la aplicación o sitios funcionen.


Algunos usos de JavaScript al dia de hoy serian:


Servidores web Validación del lado del cliente
Desarrollo de videojuegos Ventanas emergentes y cajas de texto
Elementos animados Menus dinámicos desplegables
Aplicaciones del servidor Bases de datos

Entrando más a fondo el tema, la funcionalidad de JavaScript consiste en que es un lenguaje versátil y poderoso; permitiendo crear interacciones dinámicas y enriqueciendo la experiencia del usuario en las páginas web, llevandolo a su amplia adopción y uso generalizado en la industria. JavaScript se interpreta en el navegador web, lo que significa que el código JavaScript se ejecuta directamente en el entorno del navegador sin necesidad de una etapa de compilación previa.


Ahora que hemos visto y conocido un poco más sobre el lenguaje JavaScript podemos hablar sobre una parte muy importante y fundamental lo que en programación se conoce como variables, su manejo y más importante antes de eso pues las caracteristícas que componen a JavaScript.


caracteristícas de JavaScript


JavaScript es un lenguaje interpretado debido a la forma en que se ejecuta y procesa su código. por lo que entre sus caracteristicas más importantes entran:

•Entorno de ejecución en el navegador: JavaScript principalmente se ejecuta en los navegadores web. Cuando se carga una página web que contiene código JavaScript, el navegador interpreta y ejecuta ese código directamente en su entorno de ejecución interno. Esto nos quiere decir que no se requiere un paso de compilación previo antes de ejecutar el código JavaScript.

•Análisis y ejecución línea por línea: a diferencia de los lenguajes compilados, en los que todo el código se traduce en instrucciones de máquina antes de su ejecución, JavaScript se interpreta línea por línea a medida que se encuentra. Debido a que el mmotor de JavaScript del navegador analiza y ejecuta cada línea de código a medida que se encuentra durante el proceso de carga de la página.

•Flexibiladad y facilidad de desarrollo: el enfoque interpretado de JavaScript brinda flexibilidad y facilidad de desarrollo, no es necesario compilar el código antes de probarlo, lo que le permite un desarrollo rápido y una iteración ágil. Los cambios que se realizen en el código de JavaScript se podran ver y probar sin la necesidad de un proceso de compilación.

•Plataforma independiente: el hecho de que JavaScript sea un lenguaje interpretado contribuye a su capacidad de ser ejecutado en diferentes plataformas y sistemas operativos. Los navegadores web son la plataforma principal para la ejecución de JavaScript, y los navegadores modernos han implementado sus propios motores para interpretar y ejecutar el código en diferentes sistemas.

•Fácil actualización y distribución: con JavaScript interpretado, los cambios o actualizaciones en el código se pueden implementar fácilmente, ya que los usuarios solo necesitan cargar la nueva versión del código en sus navegadores. No es necesario que los usuarios realicen una instalación manual o actualización de un programa o aplicación.


Manejo de variables en JavaScript


Las variables son un concepto fundamental en cualquier lenguaje de programación. En JavaScript, se pueden declarar variables usando las palabras clave var, const o let.

En el contexto de la programación los datos son información que usamos en nuestros programas informáticos. Un ejemplo claro de dato seria el nombre de usuario en "X"; y debido a que gran parte de la programación se trata de manipular o mostrar datos se necesita una manera de guardar y registrar esos mismos datos para poder utilizar un mismo dato a lo largo del código, una analogía útil es pensar en las variables como etiquetas para nuestros valores. En otras palabras las variables son la manera en la que los programadores le dan nombrea un valor para poder reusarlo, actualizarlo o simplemente registrarlo.


Git y GitHub


Ha llegado el momento de hablar sobre Git y GitHub y comprender que son y para que nos sirven.

Git

Git es un sistema de control de versiones
que realiza un seguimiento de los cambios en los archivos.
Git es especialmente útil cuando un grupo de personas y nosotros estamos
haciendo cambios en los mismos archivos al mismo tiempo.

Para poder hacer un trabajo fluido en Git necesitamos hacer lo siguiente:

Crear una rama a partir de la copia principal de archivos
en los que nosostros (y nuestros colaboradores) puedan seguir trabajando.

Realizar modificaciones en los archivos de forma independiente y segura
en tu propia rama personal.

Dejar que Git fusione mediante combinación y de forma inteligente los cambios
específicos en la copia principal de archivos, de modo que los cambios no
afecten a las actualizaciones de otras personas.

Dejar que Git realice un seguimiento de tus cambios y los de otras personas,
por lo que todos seguiran trabajando en la versión más actualizada del proyecto.


Ahora hablaremos sobre GitHub

GitHub

GitHub es una plataforma donde puedes almacenar, compartir y trabajar junto con otros usuarios para escribir código.
Esto nos ayuda a mejorar la eficiencia y orden en el que escribimos código, al crear un repositorio nos permite hacer muchas
cosas que nos favorecen tanto a nosotros como a los colaboradores que nos ayuden.

Al crear un repositorio en Git Hub nos permite lo siguiente:

Presentar o compartir el trabajo.

Seguir y administrar los cambios en el código a lo largo del tiempo.

Dejar que otros usuarios revisen el código y realicen sugerencias para mejorarlo.

Colaborar en un proyecto compartido, sin preocuparse de que los cambios afectarán al trabajo de los colaboradores antes de que esté listo para integrarlo.

Siendo así el trabajo colaborativo, una de las características fundamentales de Github, todo esto es posible gracias al software de código abierto Git en el que se basa GitHub.

Así es como Git y GitHub se complementan pero para entenderlo más hay que saber sobre:

¿Cómo funcionan Git y GitHub juntos?

Al cargar archivos en GitHub, los almacenas en un"repositorio de Git". Esto significa que al realizar cambios (o"compromisos") en los archivos Github, Git se iniciará automáticamente para realizar un seguimiento y administrar los cambios.

Hay muchas acciones relacionadas con Git que puedes completar en GitHub directamente en el navegador, como crear un repositorio de Git, crear ramas y cargar y editar archivos.

Sin embargo, la mayoría de las personas trabajan en sus archivos localmente (en su propio ordenador), luego sincronizan continuamente estos cambios locales y todos los datos de Git relacionados, con el repositorio central "remoto" en GitHub. Hay muchas herramientas que puedes usar para hacerlo, como GitHub Desktop.

Cuando se trabaja en equipo es necesario colaborar con otros y cuando en el momento se da que todos tienen que trabajar al mismo tiempo se opta por las siguientes recomendaciones:

Extraer todos los cambios más recientes realizados por el equipo colaborador del repositorio remoto en Github.

Transferir de vuelta los cambios al mismo repositorio en Github.

En conclusión Git determina cómo combinar inteligentemente este flujo de cambios y GitHub te ayuda a administrar el flujo a través de características como "solicitudes de cambios".
